01Variables

Decision variables

Variables represent quantities to decide, usually x and y.

  • CBSE writing: Define variables clearly before equations.
  • JEE focus: Correct variables make constraints easy.
  • Common trap: Do not choose variables after writing constraints.
02Objective

Objective function

The objective is the linear expression to maximize or minimize.

  • CBSE writing: Write Maximize Z=... or Minimize Z=...
  • JEE focus: Objective line moves parallel across region.
  • Common trap: Do not confuse objective with constraints.
03Constraints

Constraints

Constraints are inequalities from limited resources or conditions.

  • CBSE writing: Include non-negativity constraints.
  • JEE focus: Boundary lines form the feasible polygon.
  • Common trap: Forgetting x>=0 and y>=0 is common.
04Graph

Feasible region

The feasible region is the common solution of all constraints.

  • CBSE writing: Shade the correct side of every boundary.
  • JEE focus: Bounded vs unbounded matters for optimum.
  • Common trap: One wrong shaded side changes answer.
05Corner

Corner point method

For bounded feasible regions, optimum occurs at a corner point.

  • CBSE writing: Make a corner table.
  • JEE focus: Check all vertices systematically.
  • Common trap: Do not test random interior points.
06Unbounded

Unbounded feasible region

An unbounded region may still have an optimum, but it must be checked.

  • CBSE writing: Use open half-plane test if needed.
  • JEE focus: Do not assume unbounded means no solution.
  • Common trap: No finite maximum is possible in some unbounded cases.
07Interpret

Interpret final answer

Final answer should name variable values and optimum value in context.

  • CBSE writing: Write a complete final sentence.
  • JEE focus: Units and integer restrictions may matter.
  • Common trap: Do not stop at Z value only.
08Table

Corner table

List every corner and substitute into objective function.

  • CBSE writing: The largest or smallest table value is the answer.
  • JEE focus: Ties can mean multiple optimal solutions.
  • Common trap: Missing a corner can miss the optimum.

Objective

Objective function

Z = ax + by

Quantity to maximize or minimize.

  • Example: Profit Z=40x+30y.
  • Trap: Objective is not a constraint.
Constraint

Linear constraint

ax + by <= c or ax + by >= c

Represents a resource or condition.

  • Example: 2x+y<=10.
  • Trap: Inequality direction matters.
Nonnegative

Non-negativity

x >= 0, y >= 0

Quantities usually cannot be negative.

  • Example: Products made cannot be negative.
  • Trap: Often forgotten.
Corner

Corner point rule

Optimum occurs at a vertex of feasible region

For bounded linear programming problems.

  • Example: Evaluate Z at every corner.
  • Trap: Interior testing is unnecessary.
Boundary

Boundary line

Replace inequality by equality to draw line

Graph ax+by=c first.

  • Example: Draw 2x+y=10 for 2x+y<=10.
  • Trap: Then choose correct side.
Test

Origin test

Substitute (0,0) to choose side when line not through origin

Quick half-plane check.

  • Example: 0<=10 means origin side works.
  • Trap: Cannot use origin if line passes through origin without care.
Tie

Multiple optimum

Same optimum at adjacent corners => every point on joining segment is optimal

Objective line parallel to boundary.

  • Example: Two corner values tie.
  • Trap: State infinitely many optimal solutions if required.
Unbounded

Unbounded warning

Unbounded feasible region may have no finite max/min

Check direction of objective.

  • Example: Z=x+y on first quadrant has no maximum.
  • Trap: Still may have minimum.

Derivation bank

Proofs that can be reproduced in exams.

Open a proof, follow the steps, and check the final result.

Why corner points matter
  1. Objective Z=ax+by has parallel level lines.
  2. Move the level line across feasible region.
  3. Last or first contact with a bounded polygon occurs at a vertex or edge.
  4. Therefore checking vertices gives optimum.

Final: Optimum occurs at a corner for bounded feasible polygon.

Shortcut: Linear function changes along straight lines.

Warning: Assumes linear objective and polygonal feasible region.

Graphing an inequality
  1. Replace inequality by equality to draw boundary line.
  2. Choose a test point not on the line.
  3. Substitute it in the inequality.
  4. Shade the side that satisfies the inequality.

Final: Boundary plus correct half-plane.

Shortcut: Draw equality line first.

Warning: Test point decides side.

Non-negativity constraints
  1. Decision variables represent amounts.
  2. Amounts like items, hours, or units cannot be negative.
  3. So x and y are at least zero.
  4. This restricts the graph to first quadrant.

Final: x>=0,y>=0 restrict region to first quadrant.

Shortcut: Quantities cannot be negative.

Warning: Some pure math examples may differ.

Multiple optimal solutions
  1. Evaluate Z at all corners.
  2. If two adjacent corners have the same best value, the objective is constant along their segment.
  3. Every point on that segment is feasible.
  4. Hence multiple optimal solutions exist.

Final: Tie on adjacent vertices gives all points on the edge.

Shortcut: Objective line overlaps boundary edge.

Warning: Vertices must be adjacent.

BasicsEasy

1. What is an objective function?

Answer: The linear expression to maximize or minimize

It is usually written as Z=ax+by.

Trap: It is not a constraint.

ConstraintEasy

2. Why add x>=0 and y>=0 in many LPPs?

Answer: Quantities cannot be negative

Products, hours, or units are non-negative.

Trap: Do not forget first quadrant constraints.

GraphMedium

3. How do you draw 2x+y<=10?

Answer: Draw 2x+y=10 and shade the side containing origin

Origin gives 0<=10, so origin side is feasible.

Trap: Use a test point.

CornerMedium

4. Where does optimum occur in bounded LPP?

Answer: At a corner point

Corner point theorem.

Trap: Do not test random interior points.

TableMedium

5. What should a corner table contain?

Answer: Corners and objective values

Substitute every vertex into Z.

Trap: Missing one corner can change answer.

UnboundedHard

6. Does unbounded feasible region always mean no optimum?

Answer: No

It may still have a finite minimum or maximum depending on objective direction.

Trap: Check objective behavior.

TieHard

7. What if two adjacent corners give same maximum?

Answer: Multiple optimal solutions on the segment

The objective line is parallel to that edge.

Trap: Corners must be adjacent for segment conclusion.

WordMedium

8. First step in a word LPP?

Answer: Define decision variables

Let x and y represent quantities.

Trap: Do not jump directly to equations.

ObjectiveMedium

9. If profit per unit is 40 and 30, write objective.

Answer: Maximize Z=40x+30y

Multiply profit by units.

Trap: Use correct coefficients.

ConstraintMedium

10. Resource limit usually creates which inequality?

Answer: <= type

Use at most available resource.

Trap: Minimum requirement uses >=.

CBSEMedium

11. Why is graph important in LPP?

Answer: It identifies feasible region and corner points

The solution is found visually and by corner table.

Trap: Algebra alone can miss shaded side.

JEEHard

12. When can the feasible region be empty?

Answer: When constraints have no common solution

Half-planes do not overlap.

Trap: Empty region means infeasible problem.

Core formulas

  • Objective function: Z = ax + by
  • Linear constraint: ax + by <= c or ax + by >= c
  • Non-negativity: x >= 0, y >= 0
  • Corner point rule: Optimum occurs at a vertex of feasible region
  • Boundary line: Replace inequality by equality to draw line
  • Origin test: Substitute (0,0) to choose side when line not through origin

Common traps

  • Define decision variables clearly.
  • Include x>=0 and y>=0 when quantities are non-negative.
  • Draw boundary line before shading.
  • Test a point to choose the correct half-plane.
  • Evaluate every corner point.
  • State answer in the context of the word problem.
